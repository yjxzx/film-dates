# Film Dates — 外国电影中国上映信息站 · 设计文档

**日期：** 2026-06-08
**状态：** 已定稿

---

## 项目概述

一个记录外国电影在中国大陆上映/即将上映信息的个人网站。灵感来自 [yuc.wiki](https://yuc.wiki)（長門番堂），但聚焦于**所有外国电影**（非国产片）在中国的院线档期。

**核心理念：** 个人手工维护 + 社区 PR 贡献 + 极简信息展示。

---

## 视觉设计

### 风格
- **复古印刷**（Retro Print）：纸质感底纹、粗边框、衬线标题 + 等宽辅助字体
- **新闻纸配色**：米白底（`#f5f0e8`）+ 纯黑墨色（`#1a1a1a`），极简克制，无彩色

### 布局
- **卡片网格**：2 列标准卡片，每卡包含海报缩略图、类型标签、上映日期
- **页面结构**：首页 = 当前年份（2026），顶部年份切换器，侧栏国家筛选

### 字体
- 标题：Georgia / 衬线体
- 正文/标签：等宽字体（monospace）
- 中文：系统默认中文字体

## 页面结构

```
/（首页 = 2026年时间线）
├── 顶部：搜索框 + 年份切换器
├── 侧栏：国家筛选
└── 主区：按月分组的时间线
    ├── 6月 ← 当前月份，默认展开
    │   ├── 电影卡片
    │   └── 电影卡片
    ├── 7月
    │   └── ...
    ├── 8月
    │   └── ...
    └── ...
各月按时间顺序排列，当月/下月默认突出，已过月份可折叠

/updates（更新日志）
├── 按日期倒序的变更列表
├── 每条：日期 + 变更类型（新增/更新）+ 片名 + 变更摘要
└── 数据来源：Git 提交历史中匹配 `[新增]` / `[更新]` 的 commit

/about
└── 关于本站 + 如何贡献（GitHub PR 流程说明）
```

### 更新日志（/updates）

- 构建时解析 Git 提交历史
- 匹配 `[新增] 片名` 或 `[更新] 片名 - 变更内容` 格式的 commit
- 按日期倒序展示，每天一个分组
- 影迷可以订阅 RSS / 定期访问查看最新动态
- 天然契合 Git 工作流，无需额外维护

- 首页即当前年份的月度时间线
- 每月一个区块，标题如「6月」，下方为当月电影卡片网格（2列）
- **当月和即将到来的月份**默认高亮展开
- **已过去的月份**可折叠，点击展开回顾
- 若某月无电影，显示「暂无定档影片」
- 电影卡片沿用标准型设计（海报位 + 引进状态 + 中文片名 + 外文原名 + 国家 + 类型标签 + 上映状态 + 上映日期）

### 核心使用场景

用户打开首页 → 直接看到「本月 + 下月」有哪些外国电影上映。不需要任何点击。

### 筛选与搜索
- 搜索框 → 客户端实时过滤，匹配中文片名、外文原名，输入即过滤
- 年份切换 → 切换至对应年份的时间线
- 国家筛选 → 在所有月份内过滤，隐藏不匹配的卡片和空的月份
- 三者可组合使用

## 数据模型

每条电影一个 MDX 文件：

```mdx
---
slug: toy-story-5
title: 玩具总动员5
originalTitle: Toy Story 5
country:
  - 美国
genres:
  - 动画
  - 冒险
importStatus: 确认引进
releaseStatus: 已定档
releaseDate: 2026-06-19
distributor: 中国电影集团
sources:
  - name: 国家电影局
    url: https://www.chinafilm.gov.cn
  - name: 官方微博
    url: https://weibo.com/...
timeline:
  - date: 2026-01-15
    event: 海外官宣定档6月19日
  - date: 2026-04-20
    event: 确认引进
  - date: 2026-05-01
    event: 宣布中国内地同步上映
---

<!-- 可选：Markdown 正文，用于备注/补充说明 -->
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|:---:|------|
| `slug` | string | ✅ | 唯一标识符，英文片名 kebab-case（如 `toy-story-5`），永久不变 |
| `title` | string | ✅ | 中国大陆官方译名 |
| `originalTitle` | string | ✅ | 原文片名 |
| `country` | string[] | ✅ | 制片国家，支持多国合拍（如 `[美国, 英国]`、`[日本, 韩国]`） |
| `genres` | string[] | ✅ | 类型标签（科幻/动作/动画/剧情/惊悚/喜剧/爱情/恐怖等） |
| `importStatus` | enum | ✅ | `未确认` / `引进传闻` / `确认引进` / `确认不上映` |
| `releaseStatus` | enum | ✅ | `待定档` / `已定档` / `热映中` / `已下映` |
| `releaseDate` | date | | 确切上映日期（`YYYY-MM-DD`），定档后填写，可为空 |
| `releasePeriod` | string | | 大致上映时间，未定档时填写（如 `2026-07`、`2026年夏季`），可为空 |
| `distributor` | string | | 中国大陆发行方（如 中国电影集团、华夏、华桦传媒、阿里影业等） |
| `sources` | object[] | | 信息来源，每条含 `name` 和 `url`。第一条为主要来源，页面默认只显示主要来源，其余点击展开 |
| `timeline` | object[] | | 历史记录，每条含 `date`（日期）和 `event`（事件描述），按时间升序。记录引进/定档/改档全过程 |

**注意：** `releaseDate` 和 `releasePeriod` 至少填一个。已定档填 `releaseDate`，未定档填 `releasePeriod`。

**最后更新时间：** 不在 frontmatter 中手动维护。构建时通过 `git log` 读取每个文件的最后修改时间，自动生成，永不出错。

### 两个状态维度

**引进状态**（能否在中国上映）：

```
未确认 → 引进传闻 → 确认引进
                  ↘ 确认不上映
```

**上映状态**（在院线的阶段）：

```
待定档 → 已定档 → 热映中 → 已下映
```

两维度独立但相关。例如：`确认引进 + 待定档` 表示已过审但没定具体日期；`引进传闻 + 未确认` 表示仅有引进传闻。

## 收录范围

### A 类 — 必须收录
满足任一条件即收录：`确认引进`、`已定档`、`热映中`、`已下映`

### B 类 — 优先观察（满足任一即收录）
- 六大制片厂出品：迪士尼、华纳、环球、派拉蒙、索尼、传奇
- 日本动画剧场版（大规模上映）
- 全球票房前 50 的影片
- 知名 IP 续作 / 衍生电影

### C 类 — 暂不收录
- 仅电影节 / 影展放映
- 纯流媒体发行（未进院线）
- 独立电影（全国排片 < 1000 场）
- 地区限定发行（仅部分城市上映）

## 贡献流程

1. Fork 仓库
2. 在 `content/movies/` 下新建/修改 `.mdx` 文件（平铺，不分子目录）
3. 提交 PR，标题格式：`[新增] 片名` 或 `[更新] 片名 - 变更内容`
4. 维护者审核合并 → Vercel 自动部署

**文件名规范：** 用 `slug.mdx`（如 `toy-story-5.mdx`）。文件无需按年份分目录，年份由 `releaseDate` / `releasePeriod` 字段推导，改档期无需移动文件。
